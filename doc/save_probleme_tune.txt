Étapes pour résoudre
1. Vérifier l’emplacement de postgresql.conf
Par défaut avec PostgreSQL 18 installé via l’installeur officiel :
C:\Program Files\PostgreSQL\18\data\postgresql.conf


Tu peux confirmer avec la commande SQL :
SHOW config_file;


Cela renvoie le chemin exact du fichier.

2. Lancer timescaledb-tune.exe avec le chemin complet
Depuis ton dossier où se trouve timescaledb-tune.exe, exécute :
.\timescaledb-tune.exe --conf-path="C:\Program Files\PostgreSQL\18\data\postgresql.conf"


⚠️ Attention :
- Mets bien les guillemets autour du chemin (à cause des espaces dans Program Files).
- Lance la console en Administrateur pour que l’outil puisse modifier le fichier.

3. Vérifier que l’extension est bien installée
Après avoir appliqué timescaledb-tune, redémarre PostgreSQL :
net stop postgresql-x64-18
net start postgresql-x64-18


Puis dans psql :
CREATE EXTENSION timescaledb;
\dx
